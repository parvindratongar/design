Desktop Application for Currency Exchange

We have 4 parts in this project:

  • Admin Panel
  • Customer web application (Corporate level)
  • Bulk Upload and
  • Manual Transaction

In this Case-Study we are focusing only 2nd one Customer Web Application. I would like to introduce about the application first, It is a currency exchange application. A corporate/Individual can signup and assign users with the same corporate and make as many wallets for every supported currency. Then Individual user of one of the corporates can perform the transaction.

Project overview: Our company has developed a desktop application for currency exchange, which allows users to exchange major currencies such as US dollar, Euro, and Indian Rupee, among others. The application have two fees, charged during each exchange: FX fee and Service fee. The FX fee is the exchange rate difference between the currencies, while the Service fee is the fee charged by the company for providing the service.

Overview

  1. Calculating the currency on Real-time and complete the transaction with that calculation was the challenge, because currency changed frequently.
  2. Transparency in Fees.
  3. User-friendly Interface: There was calculations and fees, It was a challenge to show all the calculations in a very simple and clean UI.
  4. Secure Transactions.
  5. Customer Support.

Individuals and businesses often face challenges when it comes to exchanging currencies, especially when dealing with multiple currencies. They may not have access to real-time exchange rates or may be unaware of the fees associated with the exchange. Moreover, the process may not be user-friendly, and users may face security concerns while exchanging currencies. We focused to create these functionalities.

Goals

FinTech application always a challenge to design to take care of every aspect as there are many calculations and fees. So make it simple and understandable to every user we need to manage real time conversion rate and fees visible separately for easy understand.

Goal 1 (Currency Exchange): We are exchanging the currencies on real time so we make sure to complete the flow on given time. If time goes up user needs to get new exchange rate to complete the transaction.

Goal 2 (Debit the currency to Beneficiary): We need to make another system to transfer the currency to a Beneficiary. These are also two types of beneficiary, Same currency and Different currency. Same currency will apply only transfer fee but Different currency will apply both FX fee and Transfer fee.

Goal 3 (Credit the currency from Beneficiary): While credit the currency we need to pay only transaction fee in both conditions either it is a Same currency or Different currency.
